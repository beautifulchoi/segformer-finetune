The whole procedure

One controlled path from raw data to usable predictions.

Every stage has a clear input and output. The manifest assigns the split, the dataset validates the pair, the converter protects the pretrained weights, and inference writes class-ID masks.

Key rule:camera tokens such as cam1, cam2, and cam3 are used only for statistics. They never decide pairing or the train/val/test split. On narrow screens, scroll the diagram horizontally to see the complete flow.
Procedure

Six steps, each with a concrete artifact.

Use the sample COCO128-Seg run as a reference, then replace the data preparation step with your own image-mask export.

01

Prepare image and mask pairs

Place every source image and its single-channel mask in a reproducible dataset directory. The sample script downloads COCO128-Seg, rasterizes YOLO polygons, and writes class IDs.

  • Image and mask dimensions must match.
  • Mask values must be {0, 1} or {0, 255}.
  • Use the CSV, not filename conventions, to define pairing.
02

Create and validate the CSV manifest

The manifest contains image, mask, and split columns. Paths may be absolute or relative to the manifest directory or configured data root.

  • Rows are assigned to train, val, or test.
  • Missing files, duplicate pairs, invalid masks, empty splits, and shape mismatches fail early.
  • The dataset prints split counts and optional camera statistics.
03

Convert the Hugging Face checkpoint

The converter maps the B0 backbone and compatible SegFormer decoder weights from nvidia/segformer-b0-finetuned-ade-512-512 into the local MMSeg key format.

  • Backbone loading coverage is checked before saving.
  • ADE-150 classifier weights are skipped because this task has two classes.
  • The serialized tensors are compared after saving.
04

Configure the binary segmentation model

The custom config keeps the original MMSeg training flow while swapping in CsvBinaryDataset and LoadBinaryAnnotations.

  • SegFormer-B0 decode head uses num_classes=2.
  • Images are resized to 512 × 512 with ImageNet normalization.
  • Batch size 1 and zero workers make the one-GPU smoke test portable.
05

Train one epoch and validate

Run the epoch-based smoke test on one visible GPU. The runner saves an epoch checkpoint and evaluates the validation split at the end of the epoch.

  • Expected checkpoint: work_dirs/segformer_b0_binary_smoke/epoch_1.pth.
  • The verified sample run reported validation mIoU 51.51% and mDice 67.87%.
  • Before longer training, confirm the loss decreases and the masks look plausible.
06

Evaluate the holdout and run inference

Test evaluation reads only CSV rows with split=test. The root-level inference script accepts an image, directory, or CSV and writes class-ID masks plus original-resolution overlays.

  • Raw masks contain only 0 and 1.
  • CSV inference reports aggregate IoU and Dice when masks are present.
  • Output names include an index to avoid collisions.
What changed

What was changed and what was added in this fork.

The official SegFormer architecture and MMSegmentation training flow remain the foundation. The additions make custom binary segmentation reproducible and inspectable.

Original files changed

Only compatibility and registration points were edited in the upstream codebase.

  • Registered the CSV dataset and binary annotation loader.
  • Added PyTorch 2.8 / legacy MMCV compatibility shims.
  • Removed optional decode-head imports that require unavailable MMCV compiled ops.
  • Updated legacy NumPy float aliases used by evaluation metrics.

New workflow files

These files implement the custom dataset path without replacing the official configs.

  • CSV dataset, binary mask loader, and tests.
  • COCO128-Seg preparation and validation script.
  • Hugging Face to MMSeg checkpoint converter and report.
  • Two-class training config and root-level inference script.

New documentation and evidence

The repository now explains and records the verified path.

  • Top-of-file README notice for custom-dataset fine-tuning.
  • RUN_REPORT.md with setup commands, metrics, and compatibility notes.
  • This visual guideline with the pipeline diagram and operating procedure.
  • Inference masks and overlays from the five-image test split.

Intentionally unchanged

The original ADE20K configs, model architecture, and upstream training entry points remain available. The sample data, pretrained weights, and work directories stay ignored and local.

Run it

Commands for the verified sample workflow.

Run these from the repository root with the existing CUDA-enabled environment. Replace paths after your own CSV and checkpoint are ready.

Prepare the sample CSV

Creates a deterministic 20 / 5 / 5 train-val-test split.

python tools/prepare_coco_csv.py \
  --output-dir data/coco_binary \
  --cache-dir data/cache \
  --max-images 30 \
  --seed 20260719

Convert pretrained weights

Writes an MMSeg checkpoint with the two-class head reinitialized.

python tools/convert_hf_segformer_to_mmseg.py \
  --config local_configs/segformer_b0_binary_csv.py \
  --output pretrained/segformer_b0_ade512_mmseg.pth \
  --report pretrained/segformer_b0_ade512_mmseg.json

Train one epoch on GPU 0

The custom config already points to the CSV and converted checkpoint.

CUDA_VISIBLE_DEVICES=0 python tools/train.py \
  local_configs/segformer_b0_binary_csv.py \
  --work-dir work_dirs/segformer_b0_binary_smoke \
  --seed 20260719 --deterministic

Evaluate the test split

Uses only rows with split=test.

CUDA_VISIBLE_DEVICES=0 python tools/test.py \
  local_configs/segformer_b0_binary_csv.py \
  work_dirs/segformer_b0_binary_smoke/epoch_1.pth \
  --eval mIoU mDice

Run reusable inference

Writes masks and overlays into the selected output directory.

CUDA_VISIBLE_DEVICES=0 python inference.py \
  --config local_configs/segformer_b0_binary_csv.py \
  --checkpoint work_dirs/segformer_b0_binary_smoke/epoch_1.pth \
  --input data/coco_binary/manifest.csv \
  --output-dir outputs/inference_test \
  --device cuda:0 --split test
Sample result: the 30-image run completed one epoch, evaluated five test rows at mIoU 41.11% and mDice 55.79%, and produced five masks and five overlays. These numbers are a smoke-test reference, not a target for a new dataset.
Use your own data

What changes, and what stays fixed.

The model and training flow stay close to the official repository. Your work is concentrated in the manifest and the data preparation step.

Change for your dataset

Set csv_path, optionally set data_root, and keep the configured column names aligned with your CSV. If needed, customize the preparation script or provide your own manifest.

Keep the contract

Use one image per mask, binary class IDs, explicit split labels, matching dimensions, and non-empty train, val, and test rows. Do not infer split membership from camera names.

Scale beyond the smoke test

Increase the dataset, tune crop and augmentation choices, train for more epochs, and keep a truly untouched test split. Review qualitative masks alongside mIoU and Dice.

Before trusting a checkpoint

Confirm the logits have two channels, the conversion report has full backbone coverage, the checkpoint reloads, and inference outputs contain only class IDs 0 and 1.

Repository map

Where each part of the workflow lives.

tools/prepare_coco_csv.py

Downloads the sample data, writes binary masks, creates the manifest, and validates every row.

mmseg/datasets/csv_binary.py

Loads explicit image-mask pairs from the CSV and exposes the train, val, and test splits.

tools/convert_hf_segformer_to_mmseg.py

Maps the Hugging Face state dict and records coverage, skipped keys, and mismatches.

local_configs/segformer_b0_binary_csv.py

Defines the two-class model, CSV datasets, preprocessing, optimizer, and one-epoch runner.

inference.py

Runs a checkpoint on one image, a directory, or a CSV split and writes masks and overlays.

RUN_REPORT.md

Contains the exact setup, training, evaluation, inference commands, metrics, and compatibility notes.
