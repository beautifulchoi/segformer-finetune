# SegFormer B0 CSV Binary-Segmentation Run

Date: 2026-07-19

## Dataset

- Source: COCO128-Seg, downloaded from `https://github.com/ultralytics/assets/releases/download/v0.0.0/coco128-seg.zip`.
- Materialized dataset: `data/coco_binary/`.
- Manifest: `data/coco_binary/manifest.csv`.
- Selection: 30 deterministic images, seed `20260719`; 20 train, 5 val, 5 test.
- Masks: PNG class IDs `{0,1}` generated by rasterizing the source polygons. The CSV is the authoritative pairing and split manifest.

## Commands and evidence

Install the cloned repository without changing the existing Torch stack:

```bash
/home/prj/cross_platform_builder/.venv-source-cu128/bin/python -m pip install -e . --no-deps
```

Prepare and validate the CSV:

```bash
PYTHONPATH=$PWD /home/prj/cross_platform_builder/.venv-source-cu128/bin/python \
  tools/prepare_coco_csv.py --output-dir data/coco_binary --cache-dir data/cache \
  --max-images 30 --seed 20260719
```

Result: `split counts: {'train': 20, 'val': 5, 'test': 5}`. Dataset construction and sample loading passed for all three splits.

Convert and verify the Hugging Face checkpoint:

```bash
PYTHONPATH=$PWD HF_HOME=$PWD/data/huggingface \
  /home/prj/cross_platform_builder/.venv-source-cu128/bin/python \
  tools/convert_hf_segformer_to_mmseg.py \
  --config local_configs/segformer_b0_binary_csv.py \
  --output pretrained/segformer_b0_ade512_mmseg.pth \
  --report pretrained/segformer_b0_ade512_mmseg.json
```

Result: backbone coverage `1.0`, source keys used `206`, unexpected keys `0`, shape mismatches `0`; ADE classifier weights were skipped and two-class local classifier weights were reinitialized. Tensor equality after serialization passed.

Train exactly one epoch on GPU 0:

```bash
PYTHONPATH=$PWD CUDA_VISIBLE_DEVICES=0 \
  /home/prj/cross_platform_builder/.venv-source-cu128/bin/python \
  tools/train.py local_configs/segformer_b0_binary_csv.py \
  --work-dir work_dirs/segformer_b0_binary_smoke \
  --seed 20260719 --deterministic
```

Result: exit code `0`; `work_dirs/segformer_b0_binary_smoke/epoch_1.pth` and `latest.pth` were written. Built-in validation on the val split reported `mIoU 0.5151` and `mDice 0.6787`.

Evaluate the held-out test split:

```bash
PYTHONPATH=$PWD CUDA_VISIBLE_DEVICES=0 \
  /home/prj/cross_platform_builder/.venv-source-cu128/bin/python \
  tools/test.py local_configs/segformer_b0_binary_csv.py \
  work_dirs/segformer_b0_binary_smoke/epoch_1.pth \
  --eval mIoU mDice --out work_dirs/segformer_b0_binary_smoke/test_results.pkl
```

Result: test `mIoU 41.11%`, `mDice 55.79%`; five test rows were evaluated.

Run inference:

```bash
PYTHONPATH=$PWD CUDA_VISIBLE_DEVICES=0 \
  /home/prj/cross_platform_builder/.venv-source-cu128/bin/python inference.py \
  --config local_configs/segformer_b0_binary_csv.py \
  --checkpoint work_dirs/segformer_b0_binary_smoke/epoch_1.pth \
  --input data/coco_binary/manifest.csv \
  --output-dir outputs/inference_test --device cuda:0 --split test
```

Result: five raw masks and five overlays were written. Raw masks contain only `{0,1}`, overlays retain each original image resolution, filenames are collision-safe, and aggregate inference metrics were IoU `0.204020` and Dice `0.297483`.

Focused verification: `8 passed` across dataset preparation, CSV dataset/loader, model config, and inference helper tests. A direct model smoke test produced logits with shape `(1, 2, 512, 512)` on `cuda:0`.

## Compatibility changes

- Excluded optional MMCV-op-dependent decode heads from package bootstrap because the run uses the pure-Python `mmcv 1.3.0` build.
- Added a narrow PyTorch 2.8 compatibility shim for legacy SyncBatchNorm and MMCV GPU scatter behavior.
- Replaced four legacy `np.float` aliases in evaluation metrics with `np.float64` for NumPy 2.0.
